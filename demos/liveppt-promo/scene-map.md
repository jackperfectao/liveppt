# LivePPT Promo Scene Map

## Project Meta

- 项目名：LivePPT GitHub 推广动态演示
- 受众：开发者、独立创作者、技术内容运营者
- 主目标：提升项目理解速度与 Star/Fork 转化
- 主风格：ivory-luxe（可切换 champagne-editorial / emerald-reserve）

## Scene List

### Scene 1

- 目标：一句话讲清项目价值
- 标题：LivePPT：把 GitHub 项目介绍升级为可点击发布会
- 核心信息：README + 动态分镜结合，提升传播效率
- 可视元素：封面标题 + 三指标卡
- 动效：主内容右滑入场

### Scene 2

- 目标：建立痛点共识
- 标题：为什么很多项目传播效果一般？
- 核心信息：信息平铺导致“看完但记不住”
- 可视元素：痛点清单 + 风险指标
- 动效：场景切换淡入

### Scene 3

- 目标：展示解决方案
- 标题：结构、文案、主题三层解耦
- 核心信息：场景图 + 主题令牌 + 模板复用
- 可视元素：流程 bullets + 成本指标
- 动效：卡片序列进入

### Scene 4

- 目标：突出差异化
- 标题：Anti-Clone 约束，保护品牌辨识度
- 核心信息：通过 guardrails 避免同质化风格
- 可视元素：差异化策略要点 + 品牌保护指标
- 动效：强调块高亮

### Scene 5

- 目标：建立“可落地”信任
- 标题：脚本化工作流，从规划到发布
- 核心信息：plan/theme/validate/release 全链路
- 可视元素：命令要点 + 质量指标
- 动效：分层淡入

### Scene 6

- 目标：强化开源可信度
- 标题：开源准备完整，适合长期运营
- 核心信息：README、License、Roadmap、Changelog齐全
- 可视元素：文档成熟度指标
- 动效：稳定过渡

### Scene 7

- 目标：触发行动
- 标题：现在就把你的项目做成动态发布会
- 核心信息：Star / Fork / Issue 三个 CTA
- 可视元素：CTA 按钮组 + 目标指标
- 动效：按钮顺序出现
